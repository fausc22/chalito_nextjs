RESUMEN GENERAL
---------------
Se realizó una refactorización completa del frontend con migración a shadcn/ui,
implementación del módulo de Inventario y Pedidos (estático), y reorganización de la
estructura del proyecto.


CAMBIOS ESTRUCTURALES PRINCIPALES
----------------------------------

1. MIGRACIÓN A SHADCN/UI
   - Se agregó shadcn/ui como sistema de componentes base
   - Componentes instalados en /components/ui/ (17 componentes)
   - Configuración: components.json, jsconfig.json
   - Archivo SHADCN_UI_GUIA.md con guía de uso
   - archivo componentes-demo.jsx con componentes de shadcn

2. REORGANIZACIÓN DE CARPETAS
   - /components/
     ├── auth/           → LoginForm y ProtectedRoute
     ├── dashboard/      → InfoCards, ModuleCard
     ├── inventario/     → Artículos, Ingredientes, Categorías (tabs)
     ├── layout/         → NavBar, Footer, Layout
     └── ui/             → Componentes shadcn/ui

   - /hooks/
     - Todos renombrados de .jsx a .js
     - Nuevo: useInventario.js (manejo de estado del inventario)
     - Nuevo: use-toast.js (notificaciones shadcn)

   - /lib/              → utils.js (shadcn), formatters, validators
   - /pages/
     ├── dashboard/     → Página principal con 8 módulos
     ├── inventario/    → Módulo completo de inventario
     ├── pedidos/       → Sistema de gestión de pedidos (antes ventas/)
     └── login/         → Autenticación

3. SERVICIOS API
   - articulosService.js   → CRUD de artículos
   - categoriasService.js  → CRUD de categorías
   - ingredientesService.js → CRUD de ingredientes


MÓDULO INVENTARIO (NUEVO - COMPLETO)
-------------------------------------
Ubicación: /pages/inventario/index.jsx

Tabs implementados:
1. ARTÍCULOS
   - Listado con filtros (categoría, búsqueda, estado)
   - Formulario de creación/edición
   - Card de detalles con estadísticas
   - Tabla con paginación
   - Campos: nombre, descripción, precio, categoría, estado, imagen

2. INGREDIENTES
   - Gestión completa de ingredientes
   - Filtros por nombre y estado
   - Control de stock y unidad de medida
   - Alertas de stock bajo
   - Campos: nombre, descripción, unidad, stock, stock mínimo

3. CATEGORÍAS
   - CRUD de categorías para artículos
   - Filtros y búsqueda
   - Campos: nombre, descripción, color, icono

Características:
- Diseño compacto y profesional
- Sistema de filtros avanzado
- Paginación en todas las tablas
- Modales para crear/editar
- Notificaciones con toast
- Responsive design


MÓDULO PEDIDOS (NUEVO - COMPLETO - ESTÁTICO)
----------------------------------
Ubicación: /pages/pedidos/index.jsx

Sistema completo de gestión de pedidos en tiempo real:

Columnas:
1. RECIBIDOS       → Pedidos nuevos sin procesar
2. EN PREPARACIÓN  → Pedidos marchados a cocina

Características principales:
- Interfaz tipo Kanban con 2 columnas
- TopBar con estado del local y demora de cocina
- OrderCard compacto con toda la información del pedido
- Sistema de flujo: Recibido → Marchar → En Cocina → Listo
- Colores minimalistas

Modal: Nuevo Pedido (2 pasos)
- Paso 1: Selección de productos + carrito
  · Grid de productos por categorías
  · Carrito con extras personalizables por unidad
  · Cálculo automático de totales
- Paso 2: Datos del cliente
  · Origen: local/delivery/pickup
  · Cliente y datos de contacto
  · Dirección para delivery

Modal: Cobro
- Resumen del pedido con desglose
- Múltiples métodos de pago (efectivo, débito, crédito, transfer)
- Cálculo de vuelto
- Soporte para pago parcial/completo
- Estados: PAGADO / DEBE

Diseño:
- Altura fija con scroll interno en modales
- UI compacta optimizada para 50+ pedidos/día
- Ancho max-w-7xl (consistente con inventario)
- Paginación en columnas

CONSIDERACIONES TÉCNICAS
-------------------------

1. ESTRUCTURA DE COMPONENTES

4. API
